Java-OOP

Author: Dempsey Dong Date: 一月 8, 2024  19:00:22 Category: Java-Note

1. Java 默认的构造器

对于没有构造函数的类，Java会在编译的时候默认创造一个无参数的空构造函数。

2. Java 在编译的时候怎么找包

  1. 如果是通过完整的类名来初始化的，直接寻找。如 com.ices.hitszoj.User.
  2. 否则，通过以下方式
    • 查找同包 package 下是否有对应的类；
    • 查找 import 语句，是否导入该类；
    • 查找 java.lang 是否包含了该类。

3. Java 的变量

  1. 局部变量
    局部变量为方法内部的变量，在方法被调用的时候创建，并且只能在方法内部使用。外部不可见。
    性质

    • 方法、语句块、构造函数被调用的时候被初始化，结束以后将会被销毁；
    • 访问修饰符不可以修饰局部变量，会报错；
    • 局部变量在栈上被分配；
    • 局部变量没有默认初始化，在被声明以后必须要经过初始化才可以被使用，否则报错。

  2. 成员变量
    在类内部但不在方法内的字段，用来表示类的性质。
    性质

    • 当对象被实例化的时候，成员变量也随之被实例化；
    • 成员变量在对象被销毁的时候随之一起销毁；
    • 成员变量至少被一个函数、构造函数、代码块使用，是的外部能够获得变量的信息；
    • 修饰符可以修饰成员变量，否则默认为 包内和同类 可见；
    • 成员变量具有默认值。数值型变量的默认值是 0，布尔型变量的默认值是 false，引用类型变量的默认值是 null。变量的值可以在声明时指定，也可以在构造方法中指定。

  3. 静态变量
    通过 static 修饰的声明变量。

    • 可以通过 类名.变量名 来访问静态变量；
    • 不论该类被实例化出多少对象，类只有静态变量的一份拷贝；
    • 必须在函数和构造方法之外被声明；
    • 一般应该被声明为 常量；
    • 储存在静态存储区；
    • 静态变量和成员变量不同，其在程序运行开始就被创建，在程序结束运行的时候被销毁；
    • 可以在静态语句块中被初始化。

  4. 常量
    通过 final 修饰声明的变量。

    • 不可以被修改，一旦给定就不可以再被修改；
    • 必须是全大写来命名的；

5. native 方法

native 修饰的方法叫做 本地方法，其通过JNI实现调用其他语言实现的基础功能。目前 Java 的getClass()，hashCode()等方法就是native修饰的。

  • 失去了平台可移植性。因为被调用的函数不是通过Java语言写的，本地系统必须要有这一类函数才可以执行；
  • 对于对于更接近系统界别的代码可以通过JNI调用实现更快的语言执行速度。

6. 构造方法

  1. 没有返回值；
  2. 只能被 范围 修饰符修饰；
  3. 构造方法的调用在new的时候JVM自动调用；

7. 复制对象

  1. 通过构造函数直接复制；在构造函数里面再次以本类作为参数写一个新的构造函数，就可以传对象给构造函数，从而创造出一个字段一样的对象。

  2. 通过对对象的变量进行赋值实现；通过初始化以后的setter来操作每一个字段实现复制对象。

  3. 通过Object的clone()方法实现。

8. 抽象类

  1. 抽象类不能被实例化；
  2. 抽象类必须有 一个 或者 多个 抽象方法；
  3. entendx 扩展抽象类，子类必须要实现抽象方法；
  4. 抽象方法一定没有方法体。

好处：
- 复用抽象类代码；
- 对于不同的子类抽象方法可以进行不同的实际的适应需求的操作。

9. 接口

  1. 接口由 interface 修饰；
  2. 接口中定义的变量会自动加上 public static final 即初始化以后变量就变成了不可变的常量；
  3. 接口中的没有方法体的方法自动认为是 隐式公开抽象 方法；
  4. 从Java8开始可以在接口里面实现静态方法，但是只能通过接口的类名实现调用，子类不可以继承和调用；
  5. 从Java8开始允许默认方法default的出现，因为这样可以使得所有的子类都具有该方法而不具体的修改，否则新增一个抽象方法其所有的子类都将重写该方法才能正常运行；
  6. 接口不可以被实例化；接口可以没有任何的方法（比如序列化和克隆等）；接口定义不能有final字样，因为它出现的目的就是为了被继承和实现；
  7. Java对于抽象类的继承不支持多继承，但是接口的出现导致一个类可以同时实现多个接口的方法，从而实现了类多继承
